addLayer(layer, parentLayeropt) → {Promise}

Source:
  • Core/View.js, line 314
Add layer in viewer. The layer id must be unique. This function calls `preprocessDataLayer` of the relevant provider with this layer and set `layer.whenReady` to a promise that resolves when the preprocessing operation is done. This promise is also returned by `addLayer` allowing to chain call.
Example
// Add Color Layer
view.addLayer({
     type: 'elevation',
     id: 'iElevation',
});

// Example to add an OPENSM Layer
view.addLayer({
  type: 'color',
  protocol:   'xyz',
  id:         'OPENSM',
  fx: 2.5,
  url:  'http://b.tile.openstreetmap.fr/osmfr/${z}/${x}/${y}.png',
  format: 'image/png',
  options: {
      attribution : {
          name: 'OpenStreetMap',
          url: 'http://www.openstreetmap.org/',
      },
      tileMatrixSet: 'PM',
   },
});

// Add Elevation Layer and do something once it's ready
var layer = view.addLayer({
     type: 'elevation',
     id: 'iElevation',
}).then(() => { .... });

// One can also attach a callback to the same promise with a layer instance.
layer.whenReady.then(() => { ... });
Parameters:
Name Type Attributes Description
layer LayerOptions | Layer | GeometryLayer
parentLayer Layer <optional>
Returns:
a promise resolved with the new layer object when it is fully initialized or rejected if any error occurred.
Type
Promise

getLayers(filter) → {Array.<Layer>}

Source:
  • Core/View.js, line 389
Get all layers, with an optionnal filter applied. The filter method will be called with 2 args: - 1st: current layer - 2nd: (optional) the geometry layer to which the current layer is attached
Example
// get all layers
view.getLayers();
// get all color layers
view.getLayers(layer => layer.type === 'color');
// get all elevation layers
view.getLayers(layer => layer.type === 'elevation');
// get all geometry layers
view.getLayers(layer => layer.type === 'geometry');
// get one layer with id
view.getLayers(layer => layer.id === 'itt');
Parameters:
Name Type Description
filter function

pickObjectsAt(mouseOrEvt, radius, …where) → {Array}

Source:
  • Core/View.js, line 596
Return objects from some layers/objects3d under the mouse in this view.
Example
view.pickObjectsAt({ x, y })
view.pickObjectsAt({ x, y }, 1, 'wfsBuilding')
view.pickObjectsAt({ x, y }, 3, 'wfsBuilding', myLayer)
Parameters:
Name Type Attributes Description
mouseOrEvt Object mouse position in window coordinates (0, 0 = top-left) or MouseEvent or TouchEvent
radius number picking will happen in a circle centered on mouseOrEvt. Radius is the radius of this circle, in pixels
where * <repeatable>
 where to look for objects. Can be either: empty (= look in all layers with type == 'geometry'), layer ids or layers or a mix of all the above.
Returns:
- an array of objects. Each element contains at least an object property which is the Object3D under the cursor. Then depending on the queried layer/source, there may be additionnal properties (coming from THREE.Raycaster for instance).
Type
Array
